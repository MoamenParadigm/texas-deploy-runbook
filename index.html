-- ... execute the generated INSERT script here ...

-- After running, re-enable and re-validate the constraints:
ALTER TABLE dbo.ContactusAttachment        WITH CHECK CHECK CONSTRAINT ALL;
ALTER TABLE dbo.CareerApplicant            WITH CHECK CHECK CONSTRAINT ALL;
ALTER TABLE dbo.CareerApplicantAttachment  WITH CHECK CHECK CONSTRAINT ALL;
ALTER TABLE dbo.CareerLang                 WITH CHECK CHECK CONSTRAINT ALL;
GO
Large tables / big files: the Generate Scripts wizard can be slow and produce very large .sql files. For high-row tables (e.g. attachments), the Import and Export Wizard, BCP, or a temporary linked server are faster alternatives. The end result is the same: data lands in [NEW_LIVE_DB].

13
Environment variables — sensitive values only central
▶

Only the truly sensitive values are environment variables — connection strings, the Notifications URL and the reCAPTCHA keys. All other links, file paths and allowed origins stay hardcoded in web.config / appsettings.json. .NET maps the double underscore __ to the config : nesting. Fill the ... placeholders with the real live values on the server.

Node Admin Panel
ConnectionStrings__GlobalDatabaseContainer = ...   # live [DB_NODE_CORE] connection string
Node API
ConnectionStrings__GlobalDatabaseContainer = ...   # live [DB_NODE_CORE] connection string
Texas Admin Panel
ConnectionStrings__DatabaseContainer = ...   # live [DB_MAIN] connection string
Texas API
ConnectionStrings__TexasInformativeDB = ...                 # live [DB_MAIN] connection string
ExternalServicesUrls__Notifications   = https://[NOTIFICATIONS_URL]
Recaptcha__SiteKey                    = ...                 # reCAPTCHA site key (code)
Recaptcha__SecretKey                  = ...                 # reCAPTCHA secret key (code)
JwtSettings__NotificationsKey         = ...                 # JWT notifications key (secret)
PowerShell — set machine-level (run as admin), then restart app pools
$vars = @{
  "ConnectionStrings__GlobalDatabaseContainer" = "..."
  "ConnectionStrings__DatabaseContainer"       = "..."
  "ConnectionStrings__TexasInformativeDB"      = "..."
  "ExternalServicesUrls__Notifications"        = "https://[NOTIFICATIONS_URL]"
  "Recaptcha__SiteKey"                         = "..."
  "Recaptcha__SecretKey"                       = "..."
  "JwtSettings__NotificationsKey"              = "..."
}
foreach ($k in $vars.Keys) {
  [Environment]::SetEnvironmentVariable($k, $vars[$k], "Machine")
}
# then recycle the app pools / restart the sites
web.config vs appsettings.json: the Texas API (ASP.NET Core) reads __ env vars automatically. The classic ASP.NET apps (Node Admin Panel, Node API, Texas Admin Panel) need the Microsoft.Configuration.ConfigurationBuilders.Environment config builder (or read the env var in code) for web.config to pick them up. Either way you can scope a variable to one app via IIS Application Pool → Advanced Settings → Environment Variables, then recycle the pool.
